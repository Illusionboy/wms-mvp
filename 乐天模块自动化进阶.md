# 乐天模块自动化进阶

以下为全自动化实现步骤，可立即实现1和2，WMS服务后台登录乐天生成快递解析的CSV文件，并提供下载；第3步以后的自动化有一些安全隐患和暗坑，所以需要和你探讨；第4步是我们操作人员每天花时间最长的步骤，要不停复制-搜索-填写快递单号、发送时间-确认，如果这一步能实现，那真太感谢了

## 2. 自动登录乐天下载数据

1. 登录RMS -> 登录乐天邮箱
2. データーダンロー　页面
3. 下载近10天待发送（発送待ち）的订单（
4. 乐天快递生成程序作为后台一个服务，解析生成各快递平台需要的CSV数据，文件名打上标签：时间-店铺-快递公司
5. 周一到周六早上9点自动运行，然后放到某个地址，网店操作员可以直接下载解析好的CSV文件

注：

1. 我们公司有两个乐天店铺，所以流程得走两次
2. 每个店铺要登录三个账号：1. RMS账号；2. 乐天账号； 3. Data Download账号。RMS账号经常会因为安全问题修改，你看用什么方式保存账号密码比较优雅并方便
3. 自动执行乐天采购服务，不过也需要操作员点确认出库。

## 3. 自动登录各快递公司网站，通过批量导入上一步生成的解析文件，生成真实快递单据

暗坑：

1. 存在地址过长的问题，目前流程是由我们手动将公寓或者部门地址移到下一栏
2. 邮编和地址不匹配问题
3. 邮局快递生成每一单都需要手动点支付，不知道自动化是否可行并可靠

## 4. 快递单号回填到乐天RMS中

目前流程

1. 我们通过生成的快递单（PDF文件），复制单号中的寄送地址的名字或者电话
2. 打开RMS的顾客管理页面，选择届き先名前（電話番号），将上一条复制的名字（电话）粘贴到搜索栏，找到该客户等待发送的订单，点击进去
3. 回到PDF，复制快递单号，粘贴到相应的输入框，时间点选今天，再点击最上方输入内容确认

暗坑：

1. 有时候一个客人下了两个订单，但是我们的乐天快递生成服务已经合并订单生成一张快递单；找到客人后，订单中不仅有等待发送的，还有取消、到货、售后等可能，我们只要等待发送的
2. 人操作PDF时，复制是方便的；如果是程序可能会有些复杂吧？但是都是统一的格式，能否顺利解析？
3. 一般我们在这个步骤也是确认商品JAN和数量是否解析的正确，目前快递单生成程序版本2还算没出过问题，不过后台运营人员在添加新商品时可能没按规范来，可能导致生成快递单的数量和实际下单数量有出入。
